# 가족 협업 MVP 1차 운영 배포 체크리스트

가족 그룹 만들기, 돌봄 대상 등록·수정·삭제, 보호자 초대·멤버 관리, 주 보호자 바꾸기, 내가 그룹 나가기, 그룹 삭제까지 **MVP 1차**를 배포하기 전에 **운영·QA 담당자가 화면을 따라가며** 확인할 때 쓰는 문서입니다.  
(앱 코드를 바꾸지 않고, **스테이징 또는 운영 환경**에서 점검한다는 전제입니다.)

---

## 1. 배포 전 준비

- [ ] **점검용 계정 2개**를 준비합니다. (예: A = 그룹을 만드는 사람, B = 초대받는 사람) 서로 **다른 이메일**이어야 합니다.
- [ ] **스테이징 주소**와 **운영 주소**를 구분해 적어 둡니다. 점검은 보통 **스테이징에서 먼저** 합니다.
- [ ] **로그인·회원가입**이 정상인지 한 번 확인합니다. (비밀번호 재설정 메일 등)
- [ ] **초대 링크**를 열 때 사용하는 **사이트 주소(도메인)**가 배포된 앱 주소와 같은지 확인합니다. (잘못된 주소면 수락 화면이 안 나올 수 있어요.)
- [ ] 개발팀에 **데이터베이스(SQL) 마이그레이션 적용 여부**를 확인합니다. (아래 2절 참고)
- [ ] 그룹 삭제는 **되돌리기 어렵습니다**. 점검 후 테스트 데이터를 지울지, 백업 정책이 있는지 팀과 짧게 맞춥니다.

---

## 2. 마이그레이션 확인

데이터베이스에 필요한 SQL이 **순서대로** 적용되어 있어야 앱이 동작합니다. 번호가 빠지면 **버튼을 눌러도 오류**가 나거나 메뉴가 안 보일 수 있습니다.

### 2.1 개발팀에 확인할 것

- [ ] 스테이징·운영 DB에 **`001`부터 `013`까지** 마이그레이션이 빠짐없이 적용되었는지 요청합니다.
- [ ] 배포 직후 **가족 그룹 만들기** 한 번이 되는지로 1차 이상이 적용됐는지 간단히 봅니다. (004 이후가 없으면 그룹 생성이 안 될 수 있어요.)

## 3. 정상 동작 점검

**권장 순서**대로 진행합니다. 상단 **「가족 협업 · 현재 상태」** 요약 카드가 있으면, 숫자·문구가 아래와 맞는지 같이 보면 좋습니다.

### 3.1 가족 그룹 만들기 (계정 A)

- [ ] A로 로그인 → **온보딩** 또는 안내에 따라 **가족 그룹 만들기**를 완료합니다.
- [ ] **보여야 할 것:** 보호자 대시보드에 **내 가족 그룹** 카드에 그룹 이름, 등록일(또는 생성 완료 표시)이 보입니다.
- [ ] **상단 요약:** 가족 그룹 **있음**, 돌봄 대상 **없음**, 보호자 **1명** 정도로 보이면 정상에 가깝습니다.

### 3.2 돌봄 대상(첫 등록) 등록 (계정 A)

- [ ] **온보딩**에서 돌봄 대상 이름을 입력하고 저장합니다.
- [ ] **보여야 할 것:** 대시보드 **「돌봄 대상(첫 등록)」**에 이름이 보이고, **주 보호자**에 A(또는 본인 표시)가 보입니다.
- [ ] **이름·비고 수정** 링크로 들어가 수정한 뒤, 다시 대시보드에 반영되는지 확인합니다.

### 3.3 다른 보호자 초대 (계정 A)

- [ ] **다른 보호자 초대**에서 이메일(B의 이메일)·관계를 넣고 **링크 만들기**를 누릅니다.
- [ ] **보여야 할 것:** 링크가 보이거나 복사할 수 있고, **보호자 초대 현황**에 **대기중**이 보입니다.

### 3.4 초대 수락 (계정 B)

- [ ] B로 **같은 이메일**로 가입(또는 로그인)한 뒤, A가 준 **초대 링크**를 엽니다.
- [ ] 수락 절차를 끝냅니다.
- [ ] **보여야 할 것:** A 대시보드 **「함께하는 보호자」** 목록에 B가 보입니다. 초대 현황에서 해당 건이 **수락됨** 등으로 바뀝니다.

### 3.5 주 보호자 바꾸기 (A → B)

- [ ] 보호자가 **두 명 이상**일 때만 **주 보호자 변경** 블록이 보이는지 확인합니다.
- [ ] 새 주 보호자로 **B**를 고르고 적용합니다.
- [ ] **보여야 할 것:** 돌봄 대상 카드의 **주 보호자** 문구가 **B**로 바뀝니다. 상단 요약의 주 보호자 이름도 B로 맞는지 봅니다.

### 3.6 이전 주 보호자(A) 그룹에서 빼기

- [ ] A가 더 이상 주 보호자가 **아닌지** 확인한 뒤, A 줄의 **그룹에서 빼기**를 누릅니다. 확인 창에서 허용합니다.
- [ ] **보여야 할 것:** 목록에 **B만** 남습니다.

### 3.7 (필요 시) 보호자 한 명이 그룹 나가기

- [ ] 나가기는 **보호자가 두 명 이상**일 때만 가능합니다. B만 남았다면, 점검용으로 **C를 한 번 더 초대·수락**해 두 명 이상으로 만듭니다.
- [ ] **주 보호자가 아닌** 사람이 **이 가족 그룹에서 나가기**를 실행합니다.
- [ ] **보여야 할 것:** 나간 사람은 그 그룹 목록에서 빠지고, 앱이 안내하는 기본 화면으로 이동할 수 있습니다.

### 3.8 돌봄 대상 삭제 (그룹에 보호자 1명만 남은 뒤 등, 정책에 맞게)

- [ ] 대시보드에서 **돌봄 대상 삭제**를 누르고 확인합니다.
- [ ] **보여야 할 것:** 돌봄 대상 블록이 **아직 없음** 상태로 바뀝니다. 상단 요약도 돌봄 대상 **없음**으로 바뀝니다.

### 3.9 가족 그룹 완전 삭제 (마지막 보호자, 돌봄 대상 없을 때)

- [ ] **보호자가 본인만** 있고, **돌봄 대상이 없을 때만** 삭제 버튼이 나와야 합니다. (상단 요약에 **그룹 완전 삭제 가능: 예**가 보이는지 확인합니다.)
- [ ] **가족 그룹 완전히 삭제**를 누르고 확인합니다.
- [ ] **보여야 할 것:** 그룹 카드가 없어지거나, 가족 그룹이 없을 때 안내가 보입니다. 앱이 안내하는 화면으로 이동합니다.

## 4. 실패/예외 점검

아래를 **의도적으로** 시도했을 때, **에러 메시지가 보이거나** 버튼이 **안 보이거나 비활성**이면 통과로 봅니다.

### 4.1 보호자·그룹

| 시도 | 기대 |
|------|------|
| 보호자가 **한 명뿐**인데 그 사람을 **그룹에서 빼기** | 안 되어야 함. 안내 문구 확인. |
| **주 보호자**인 사람을 **그룹에서 빼기** | 안 되어야 함. 먼저 주 보호자를 넘기라는 안내가 있으면 좋음. |
| 보호자 **한 명뿐**인데 **이 그룹에서 나가기** | 안 되어야 함. |
| 내가 그룹의 **주 보호자**인데 **나가기** | 안 되어야 함. |
| 보호자가 **두 명 이상**인데 **가족 그룹 완전 삭제** | 안 되어야 함(버튼이 안 보이거나 눌러도 실패). |
| **돌봄 대상이 있는데** 그룹 완전 삭제 | 안 되어야 함. |

### 4.2 초대

| 시도 | 기대 |
|------|------|
| 초대할 때 적은 이메일이 **아닌 다른 이메일** 계정으로 수락 | 실패해야 함. |
| **기한이 지난** 초대 링크로 수락 | 실패해야 함. |

### 4.3 주 보호자

| 시도 | 기대 |
|------|------|
| 보호자 **한 명**일 때 주 보호자 변경 UI | 안 보이거나 할 수 없어야 함. |

### 4.4 로그인·환경

| 시도 | 기대 |
|------|------|
| 로그아웃한 뒤 동작 | 로그인하라는 흐름으로 막혀야 함. |
| DB(SQL)가 덜 올라간 환경 | 자주 **함수가 없다**, **연결 오류** 같은 메시지가 납니다. → 2절로 돌아갑니다. |

## 5. 운영 반영 전 최종 확인

- [ ] **스테이징에서 3절·4절**을 모두 통과했는지 체크합니다.
- [ ] **운영**에 반영할 때도 마이그레이션(001~013)이 **스테이징과 같은 순서**로 적용되었는지 개발팀과 다시 한 번 확인합니다.
- [ ] **초대 링크**에 들어가는 **주소(도메인)**가 운영 앱 주소와 맞는지 확인합니다.
- [ ] **일반 사용자 계정**으로만 점검했는지 확인합니다. (관리자 전용 도구로만 되면 실제 사용자와 다를 수 있어요.)
- [ ] **그룹 삭제·돌봄 대상 삭제**는 복구가 어렵다는 점을 CS·운영 공지에 넣을지 팀에서 정합니다.
- [ ] 화면 문구(나가기, 삭제, 주 보호자 등)가 **고객 안내**와 어긋나지 않는지 짧게 봅니다.

---

## 6. 문제가 생겼을 때 먼저 볼 것

1. **지금 스테이징인지 운영인지** 주소를 다시 확인합니다.
2. **로그인한 계정**이 맞는지, 초대에 쓴 **이메일과 같은지** 확인합니다.
3. **마이그레이션 적용 여부**(2절)를 개발팀에 확인합니다. (가장 흔한 원인 중 하나입니다.)
4. **브라우저 새로고침** 또는 **다른 브라우저**에서 한 번 더 시도합니다.
5. 화면에 **빨간 안내 문구**가 있으면 그대로 캡처해 개발팀에 전달합니다.
6. 초대만 안 되면 **링크 전체 복사 여부**, **만료 시간**, **이메일 오타**를 확인합니다.
7. 그룹 삭제·나가기가 안 되면 **상단 요약 카드**의 **보호자 수·돌봄 대상 유무·주 보호자**가 조건과 맞는지 봅니다.
